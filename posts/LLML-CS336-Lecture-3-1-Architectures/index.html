RMS Norm

RMSNorm, which stands for Root Mean Square Normalization, is a layer normalization technique used in some transformer architectures. It’s a simpler and more computationally efficient alternative to traditional Layer Normalization.

The core principle of RMSNorm is to re-scale the activations of a layer based on their root mean square (RMS) value, rather than their mean and standard deviation.

For a given input vector xxx (the activations from a layer), the RMSNorm operation is defined as:

The normalization also happens in the dimensional feature.

yn,l,d=γd(xn,l,d1D∑i=1Dxn,l,i2+ϵ)y_{n,l,d} = \gamma_d \left( \frac{x_{n,l,d}}{\sqrt{\frac{1}{D} \sum_{i=1}^{D} x_{n,l,i}^2 + \epsilon}} \right) yn,l,d​=γd​​D1​∑i=1D​xn,l,i2​+ϵ​xn,l,d​​​

RMSNorm calculates the root mean square, which is essentially the L2 norm divided by the square root of the dimension. This method is simpler as it doesn’t subtract the mean, which is the most computationally expensive part of Layer Normalization. This makes RMSNorm more efficient and faster, especially in large-scale models (Runtime: Memory Movement). It normalizes the magnitude of the activations, which is often sufficient for stabilizing training.

  • also: it drops bias terms for memory and optimization stability.

SwiGLU Mathematical Formula

SwiGLU, which stands for Swish-Gated Linear Unit, is a variant of the Gated Linear Unit (GLU) family. It was introduced by Google Brain in the PaLM model and has since been adopted by other modern large language models, such as LLaMA, as the activation function within their feed-forward networks (FFN). It has been shown to outperform both ReLU and GeLU in terms of performance.

SwiGLU functions as a sub-module within the feed-forward network. Its core idea is to replace the traditional single-path activation with a two-path structure controlled by a “gate.”.

SwiGLU-based FFN:

FFN(x)=SwiGLU(xW1,xW2)W3FFN(x) = \text{SwiGLU}(xW_1, xW_2)W_3 FFN(x)=SwiGLU(xW1​,xW2​)W3​

Here, xxx is the input vector, and W1,W2,W3W_1, W_2, W_3W1​,W2​,W3​ are learnable weight matrices.

The core operation of SwiGLU is defined as:

SwiGLU(x,W1,W2)=(Swish(xW1))⊗(xW2)\text{SwiGLU}(x, W_1, W_2) = (\text{Swish}(xW_1)) \otimes (xW_2) SwiGLU(x,W1​,W2​)=(Swish(xW1​))⊗(xW2​)

Where:

  • xxx is the input vector.
  • W1W_1W1​ and W2W_2W2​ are two distinct weight matrices.
  • Swish(⋅)\text{Swish}(\cdot)Swish(⋅) is the activation function defined by:

    Swish(z)=z⋅σ(z)\text{Swish}(z) = z \cdot \sigma(z) Swish(z)=z⋅σ(z)

    where σ(z)\sigma(z)σ(z) is the Sigmoid function:

    σ(z)=11+e−z\sigma(z) = \frac{1}{1 + e^{-z}} σ(z)=1+e−z1​

  • ⊗\otimes⊗ denotes element-wise multiplication.
  1. Path 1 (Activation Path): The input vector xxx is multiplied by the weight matrix W1W_1W1​ to produce a vector z1=xW1z_1 = xW_1z1​=xW1​. This vector is then passed through the Swish activation function: Swish(z1z_1z1​).
  2. Path 2 (Gating Path): The input vector xxx is multiplied by a separate weight matrix W2W_2W2​ to produce another vector z2=xW2z_2 = xW_2z2​=xW2​.
  3. Gating: The result from Path 1, Swish(z1z_1z1​), is multiplied element-wise by the result from Path 2,z2z_2z2​.

This element-wise multiplication is the key “gating” mechanism. The second path, z2=xW2z_2 = xW_2z2​=xW2​, acts as a gate, dynamically adjusting the weight of each element in the activated vector from Path 1 based on the input xxx.

  • Dynamic Information Flow: The gating design allows the network to learn which features should be amplified or suppressed, providing a more precise way to control information flow. This adaptive gating ability helps the model better capture complex dependencies.
  • Smoother Gradients: Unlike ReLU’s hard cutoff (with a gradient of 0 or 1), the Swish function has a smooth gradient. This helps in mitigating the vanishing gradient problem in deep networks, leading to more stable optimization.

Positional Embeddings

Basic Positional Encoding

注意力机制抛弃了串行式的词元处理状态，采用并行化的设计而牺牲了序列信息，例如，在计算注意力分数的时候交换不同键值对的位置在形式上对最后的 Attention Score 没有影响，这会影响最后建模的序列性。

因此，需要加入额外的项来表示序列的顺序信息，即在输入中添加 Positional Encoding 来注入绝对的或者相对的位置信息。下面介绍一种最常见的位置编码：基于正余弦的固定位置编码。

输入 X∈Rn×d\mathbf{X} \in \mathbb{R}^{n \times d}X∈Rn×d，考虑嵌入矩阵 P∈Rn×d\mathbf{P} \in \mathbb{R}^{n \times d}P∈Rn×d，最终输出的结果是 X+P\mathbf{X} + \mathbf{P}X+P。其中：

p(i,2j)=sin⁡(i100002j/d), p(i,2j+1)=cos⁡(i100002j/d)p_{(i, 2j)} = \sin(\frac{i}{10000^{2j/d}}), \ p_{(i, 2j+1)} = \cos(\frac{i}{10000^{2j/d}}) p(i,2j)​=sin(100002j/di​), p(i,2j+1)​=cos(100002j/di​)

绝对位置信息

在二进制中，较高位的交替频率低于较低位，因此，位置编码效仿这样的设计，随着编码维度的增加（列，代表编码位置的不同维度），因此，对于不同的行，（行即代表着序列的原始位置信息），横坐标 iii 对应的位置编码矩阵在不同的列 jjj 下的值都不相同。

jjj 可以看做控制三角函数的频率信息（以 iii 为自变量），而频率不同可以很好的在一个周期性函数上表示非周期的特征，简单来说，每一个行学习到的 p[i, :] 都是不同的。

Dive Deeper into Sine Embeddings

Sine Embeddings, 或者叫做 Sinusoidal Embeddings，是一种对输入向量进行位置编码的方式，这种基于正弦函数的位置编码也是最先在 Attention is all you need 论文中提出的方法。

下文的一些摘录节选自 Transformer 升级之路，并且加上自己的注解。

位置编码重要性不必多言，在自回归模型 Attention Score 计算的过程中，其模型是全对称的，满足恒等式 f(x,y)=f(y,x)f(x,y) = f(y,x)f(x,y)=f(y,x)，这会对最终的序列预测任务造成困扰，因此，需要引入位置编码在学习相关性的基础上学习不同词元的序列特征。而这里的序列特征指的是绝对序列特征和相对序列特征。

为了打破对称性，位置编码改变了自注意力层的函数输入：

⟨Embed(x,i),Embed(y,j)⟩=⟨vx+PEi,vy+PEj⟩=⟨vx,vy⟩+⟨vx,PEj⟩+⟨PEi,vy⟩+⟨PEi,PEj⟩\begin{aligned} \langle \text{Embed}(x, i), \text{Embed}(y, j) \rangle &= \langle v_x + PE_i, v_y + PE_j \rangle \\ &= \langle v_x, v_y \rangle + \langle v_x, PE_j \rangle + \langle PE_i, v_y \rangle + \langle PE_i, PE_j \rangle \end{aligned}⟨Embed(x,i),Embed(y,j)⟩​=⟨vx​+PEi​,vy​+PEj​⟩=⟨vx​,vy​⟩+⟨vx​,PEj​⟩+⟨PEi​,vy​⟩+⟨PEi​,PEj​⟩​

⟨vx,vy⟩+⟨vx,PEj⟩+…\langle v_x, v_y \rangle + \langle v_x, PE_j \rangle + \dots⟨vx​,vy​⟩+⟨vx​,PEj​⟩+…, these are cross-terms that are not relative.

Thus, for the original traditional embeddings, in face of mountains of training data, the model tends to memorize the absolute position instead of the relative position, which would cause a rise of perplexity outside the boundary of training data.

RoPE

Intuition

Unlike the simple adding operation (vx+PE(i)v_x + \text{PE}(i)vx​+PE(i)), RoPE tries to “rotate” the vector, which could perfectly satisfy the “relative position” property.

Consider the most simple situation:

x=(x0,x1)∈R2x = (x_0, x_1) \in \mathbb{R}^2x=(x0​,x1​)∈R2, we can compute x′x'x′ with a rotation matrix:

Rθ=(cos⁡θ−sin⁡θsin⁡θcos⁡θ)R_{\theta}=\left(\begin{array}{cc} \cos \theta & -\sin \theta \\ \sin \theta & \cos \theta \end{array}\right)Rθ​=(cosθsinθ​−sinθcosθ​)

x′=Rθxx' = R_\theta x x′=Rθ​x

The rotation of angles and doc product will only focus on the relative rotation!

⟨Rαa,Rβb⟩=⟨R(α−β)a,b⟩\langle R_\alpha a, R_\beta b \rangle = \langle R_{(\alpha - \beta)} a, b \rangle ⟨Rα​a,Rβ​b⟩=⟨R(α−β)​a,b⟩

Algorithm

Consider a word vector (we define it as {q,k}=W{q,k}xm\{q,k\} = W_{\{q, k\}} x_m{q,k}=W{q,k}​xm​), then we can apply the rotation matrix to the word vector in position mmm:

{q,k}m=RΘ,md⋅{q,k}\{q, k\}_m = R_{\Theta, m}^d \cdot \{q, k\} {q,k}m​=RΘ,md​⋅{q,k}

  • Before the rotation, the vector does not contain any positional information. (only with semantic meaning)
  • After the rotation, the vector contain the positional information with the rotated angle. θm\theta_mθm​
  • The core component is the rotation matrix: RΘ,mdR_{\Theta, m}^dRΘ,md​

Of course, we know:

RΘ,m2=(cos⁡(mθj)−sin⁡(mθj)sin⁡(mθj)cos⁡(mθj))R_{\Theta, m}^2= \begin{pmatrix} \cos(m\theta_j) & -\sin(m\theta_j) \\ \sin(m\theta_j) & \cos(m\theta_j) \end{pmatrix} RΘ,m2​=(cos(mθj​)sin(mθj​)​−sin(mθj​)cos(mθj​)​)

下面的内容用中文，因为太精彩了。

为了简化，我们在后续使用复数运算来代替对二维向量的运算（这两个是等价的），并且我们考虑 ddd 是一个偶数，这样一个维度为 ddd 的高维向量就可以被切割为 d2\frac{d}{2}2d​ 个二维向量。（对于 qqq 和 kkk）

令 N=d2N = \frac{d}{2}N=2d​，则原始的高维向量就被表示为了 NNN 个复数表示：{z1,z2,…,zj,…,zN}\{z_1, z_2, \dots, z_j, \dots, z_N\}{z1​,z2​,…,zj​,…,zN​}, zj=x2j−1+x2jiz_j = x_{2j-1} + x_{2j} izj​=x2j−1​+x2j​i

则旋转的过程可以表示为：

zj′=zj⋅eimθjz_j' = z_j \cdot e^{im\theta_j} zj′​=zj​⋅eimθj​

θj=1100002(j−1)/d\theta_j = \frac{1}{10000^{2(j-1)/d}} θj​=100002(j−1)/d1​

来一点数学：

zj′=zj⋅eimθj=(x2j−1+ix2j)(cos⁡(mθj)+isin⁡(mθj))z_{j}^{\prime}=z_{j} \cdot e^{i m \theta_{j}}=\left(x_{2 j-1}+i x_{2 j}\right)\left(\cos \left(m \theta_{j}\right)+i \sin \left(m \theta_{j}\right)\right) zj′​=zj​⋅eimθj​=(x2j−1​+ix2j​)(cos(mθj​)+isin(mθj​))

zj′=(x2j−1cos⁡(mθj)−x2jsin⁡(mθj))+i(x2j−1sin⁡(mθj)+x2jcos⁡(mθj))z_{j}^{\prime}=\left(x_{2 j-1} \cos \left(m \theta_{j}\right)-x_{2 j} \sin \left(m \theta_{j}\right)\right)+i\left(x_{2 j-1} \sin \left(m \theta_{j}\right)+x_{2 j} \cos \left(m \theta_{j}\right)\right) zj′​=(x2j−1​cos(mθj​)−x2j​sin(mθj​))+i(x2j−1​sin(mθj​)+x2j​cos(mθj​))

其实，这就是一个二维向量 (x2j−1,x2j)(x_{2j-1}, x_{2j})(x2j−1​,x2j​) 在施加旋转矩阵 RΘ,m2R_{\Theta, m}^2RΘ,m2​之后的结果。

最终把向量的维度整合，{q,k}m\{q,k\}_m{q,k}m​ 的结果就是对 NNN 个二维向量分别施加旋转矩阵之后的结果，这些二维矩阵因为 j=1,2,…,Nj = 1,2,\dots, Nj=1,2,…,N 的选择取值不同因此带来旋转的角度也不同。

当然，实际过程中不会具体地做高维向量的切割和合并，而是整合成一个大的分块矩阵：

RΘ,md=(cos⁡(mθ1)−sin⁡(mθ1)00…sin⁡(mθ1)cos⁡(mθ1)00…00cos⁡(mθ2)−sin⁡(mθ2)…00sin⁡(mθ2)cos⁡(mθ2)…⋮⋮⋮⋮⋱)∈Rd×d\mathbf{R}_{\Theta,m}^d = \begin{pmatrix} \cos(m\theta_1) & -\sin(m\theta_1) & 0 & 0 & \dots \\ \sin(m\theta_1) & \cos(m\theta_1) & 0 & 0 & \dots \\ 0 & 0 & \cos(m\theta_2) & -\sin(m\theta_2) & \dots \\ 0 & 0 & \sin(m\theta_2) & \cos(m\theta_2) & \dots \\ \vdots & \vdots & \vdots & \vdots & \ddots \end{pmatrix} \in \mathbb{R}^{d \times d} RΘ,md​=​cos(mθ1​)sin(mθ1​)00⋮​−sin(mθ1​)cos(mθ1​)00⋮​00cos(mθ2​)sin(mθ2​)⋮​00−sin(mθ2​)cos(mθ2​)⋮​…………⋱​​∈Rd×d

在施加旋转位置编码之后的高维向量就携带着相对位置信息进入 Attention 模块进行参数学习。

Proof

RoPE 的精妙之处在于，经过旋转操作后的两个向量 q\mathbf{q}q 和 k\mathbf{k}k，它们的内积只依赖于它们之间的相对位置 m−nm-nm−n。

在注意力机制中，查询向量 q\mathbf{q}q 和键向量 k\mathbf{k}k 分别对应位置 mmm 和 nnn。RoPE 对它们进行旋转操作：

  • 旋转后的查询向量：qm=Rmq\mathbf{q}_m = \mathbf{R}_m \mathbf{q}qm​=Rm​q
  • 旋转后的键向量：kn=Rnk\mathbf{k}_n = \mathbf{R}_n \mathbf{k}kn​=Rn​k

它们之间的内积为：

⟨qm,kn⟩=⟨Rmq,Rnk⟩\langle \mathbf{q}_m, \mathbf{k}_n \rangle = \langle \mathbf{R}_m \mathbf{q}, \mathbf{R}_n \mathbf{k} \rangle ⟨qm​,kn​⟩=⟨Rm​q,Rn​k⟩

利用旋转矩阵的性质，即 RmT=R−m\mathbf{R}_m^T = \mathbf{R}_{-m}RmT​=R−m​，以及 RmTRn=Rn−m\mathbf{R}_m^T \mathbf{R}_n = \mathbf{R}_{n-m}RmT​Rn​=Rn−m​，我们可以得到：

⟨Rmq,Rnk⟩=⟨q,RmTRnk⟩=⟨q,Rn−mk⟩\langle \mathbf{R}_m \mathbf{q}, \mathbf{R}_n \mathbf{k} \rangle = \langle \mathbf{q}, \mathbf{R}_m^T \mathbf{R}_n \mathbf{k} \rangle = \langle \mathbf{q}, \mathbf{R}_{n-m} \mathbf{k} \rangle ⟨Rm​q,Rn​k⟩=⟨q,RmT​Rn​k⟩=⟨q,Rn−m​k⟩

这个结果表明，旋转后向量的内积（即注意力分数）只取决于相对位置 n−mn-mn−m，而不是绝对位置 mmm 和 nnn。这正是 RoPE 能够有效编码相对位置信息的关键。
