Tools Tutorial for Computer Science

封面出处

Abstract

In this article, you will learn a set of methodologies for using tools in computer science. It will teach you how to quickly get started and master the use of tools through practical approaches, while continuously improving your skills through ongoing practice.

Table of contents

Introduction

笔者今年作为刚刚入门计算机行业的小白，可谓是世界观发生了翻天覆地的变化。（这并不所言非虚）在开发者的世界中遨游探索，让我能够快速的成长。但是笔者在学习的过程中发现：工具的使用和学习成为了许多初学者的“拦路虎”，亦或者许多初学者在学习工具时仍然保持着过去固有的学习习惯，导致学习的效率非常的低下。

高考提分和使用工具，两者的底层逻辑是截然不同的：学习传统基础学科（高考）重视基础原理，在巩固基础原理的基础之上再去做延伸和拓展，但是学习工具的使用重视上手实践，根本目的在于提高效率，重视实用性。

举个例子，大学有一门很重要的数学课叫线性代数（Linear Algebra）。如果把它当做一门基础学科去学习，应该重点学习的是线性变换，线性空间等反映代数学本质原理的抽象概念和定理，但如果把它当做一个工具，应该重点学习的是如何使用Python，MATLAB等工具计算矩阵乘法，求行列式，求特征值。

底层逻辑的差异自然会带来学习方法上的不同，一味地继承传统的学习方法而不加以变通只能是低效且无用的。因此，本文立志阐释清楚学习工具使用的基本逻辑和学习流程，并且会以git的学习历程做示范。

Basic Principles

在开始今天的博客之前，笔者想先送给各位12字箴言：

提高效率，实践为王，二八法则。

  • 提高效率：这是工具使用的基本原理。
  • 实践为王：工具使用的落脚点不是纸上谈兵而是具体落地到项目中去使用。
  • 二八法则：这其实还是提高效率的另一种表达：花较短的时间学习工具20%的功能，便可以解决80%的问题。

接下来，笔者将以git的学习路径为例，向各位展示高效学习工具使用的行之有效的方法。

Overall Perception

重点解决问题：

感知新内容的基本信息&初步构建较为完整的知识网络

  • 这是什么？
    • 能否用已有的知识进行表述？
    • 可以用来解决现阶段什么样的问题？
  • 最基本的操作有哪些？
    • 重点：梳理出一条学习的脉络和线索
    • 知其然而不知其所以然即可
  • 重点在于大量的浅层的输入

Quick Start for a project

  • 借助他人使用的轮子的力量

  • 借助典型示例快速上手一个project

  • 最重要的部分

    以实战项目为最好的教材，在实践中学习代码（知识）是怎么实现的

From imitating to modifying and enhancing

  • 在精读完示例project后，可以尝试开始 模仿
  • 模仿————修改————优化
  • 不断地发现问题，解决问题！

Diving into principles

  • 在这个阶段，已经初步具备构建工程的基本能力，但仍需要进一步掌握底层原理和一些更高级更进阶的用法（自主性和创新性）
  • 可以在开源网站上学习他人的成果
  • 尝试阅读官方文件或者一些底层文件
  • 对底层原理的学习往往是枯燥的，对于工程实践来说初期永远是实践最重要，对于原理层面起到锦上添花的作用，目的在于为我们后续的个性化创新奠定基础。

Occasional Backtracking

  • 经典回读！！！
  • 1-2-3-4往往是循环的过程，在哪一步都会受挫，理性分析，适当求助，疯狂拷打ChatGPT

Conclusion and Notice

  • 该学习方法不适应于逻辑性比较强的理科
  • 该学习方法不适用于底层的基础学科
  • 该学习方法适用于已经存在一定迁移基础的新学科，不适合完全未曾涉猎的学科
